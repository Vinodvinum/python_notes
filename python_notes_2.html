Modules and Environments

Turn scripts into reusable pieces

The jump from beginner code to useful project code begins when you stop writing everything in one file and start organizing imports, modules, packages, environments, and clear entry points.

Module Mental Model

One file can teach two jobs

A Python module can be imported by another file or run directly. That is why the if __name__ == "__main__": block matters.

import json
import math
from datetime import datetime, timedelta

def build_report():
    today = datetime.now()
    return {
        "date": today.strftime("%Y-%m-%d"),
        "sqrt_16": math.sqrt(16),
        "tomorrow": (today + timedelta(days=1)).strftime("%Y-%m-%d"),
    }

def main():
    print(json.dumps(build_report(), indent=2))

if __name__ == "__main__":
    main()

That pattern is already visible in vinod.py. Importing a module should usually expose functions. Running it directly should usually start the demo or app.

Import Styles

Choose imports for clarity, not style points

Style Example When to use
Whole module import vinod Best default. Clear source. Easy to trace.
Alias import pandas as pd Common with long, standard library names used often.
Specific import from datetime import datetime Fine when a symbol is central and used repeatedly.
Avoid from module import * Creates confusion and naming collisions.

Package Thinking

Module vs package vs library

  • Module: one .py file.
  • Package: a folder containing related modules.
  • Library: a broader collection of reusable code, often installed through pip.
  • Notebook: great for exploration, weaker for long-term structure unless you refactor working code into modules.
A good habit: once a notebook cell becomes stable and useful, move the reusable part into a Python file and import it back into the notebook.

NumPy is the core numerical computing library in Python. It is used when data is mainly numeric and you want high performance, vectorized calculations, matrix-style operations, random number generation, and mathematical functions over full arrays.

Definition, why we use it, and base syntax

Point Meaning
Definition NumPy stands for Numerical Python. It provides the ndarray object for efficient numerical computation.
Main object ndarray, a multidimensional array.
Why we use it It is faster and more memory efficient than plain Python lists for numeric work.
Typical tasks Math operations, matrix operations, reshaping, random simulation, distributions, and preprocessing numeric data.

Indexing and Slicing

Select values from one-dimensional and two-dimensional arrays

arr = np.array([5, 10, 15, 20, 25])
print(arr[0])
print(arr[1:4])

matrix = np.array([[1, 2, 3], [4, 5, 6]])
print(matrix[0, 1])

NumPy indexing is similar to list indexing, but multidimensional arrays allow row and column access in one expression.

Shape and Reshape

Convert one layout into another layout

arr = np.arange(1, 13)
matrix = arr.reshape(3, 4)
first_row = matrix[0]
second_col = matrix[:, 1]

Reshaping does not change the data itself. It changes how the same data is arranged across dimensions.

Copy vs View

Very important memory concept

arr = np.array([1, 2, 3, 4])
view_arr = arr.view()
copy_arr = arr.copy()

arr[0] = 99
print(view_arr)  # changes
print(copy_arr)  # stays old
Concept Meaning
View Shares the same memory with the original array
Copy Creates separate memory
Broadcasting

How NumPy expands compatible shapes

scores = np.array([[70, 80, 90], [60, 75, 88]])
bonus = np.array([5, 5, 5])
updated = scores + bonus

Broadcasting means NumPy can stretch a smaller array across a bigger one when shapes are compatible. This removes many manual loops.

Vectorized Math

Operate on the whole array at once

arr = np.array([5, 8, 11, 14, 17])
print(arr + 2)
print(arr * 3)
print(arr ** 2)
print(np.sqrt(arr))
print(np.sin(arr))

Vectorization means doing one array operation instead of writing a loop for each element.

NumPy Distributions

Definition, idea, syntax, and common distribution types

Distribution What it models Common syntax Example use case
Normal Values clustered around a mean random.normal(loc, scale, size) Exam scores, heights, measurement error
Uniform All values equally likely in a range random.uniform(low, high, size) Random values between two limits
Binomial Number of successes in fixed trials random.binomial(n, p, size) Number of heads in repeated coin flips
Poisson Number of events in fixed interval random.poisson(lam, size) Calls per minute, arrivals per hour
Exponential Time between events random.exponential(scale, size) Waiting time between arrivals
Geometric Trials needed for first success random.geometric(p, size) Attempts before first success
Chi-square Positive skewed statistical distribution random.chisquare(df, size) Hypothesis testing and variance-related work 
from numpy import random

normal = random.normal(loc=50, scale=8, size=1000)
uniform = random.uniform(low=0, high=1, size=1000)
binomial = random.binomial(n=10, p=0.4, size=1000)
poisson = random.poisson(lam=3, size=1000)
Simple definition: a probability distribution describes how values are expected to occur. NumPy gives ready-made tools to generate sample data from those distributions.
NumPy UFuncs

Definition, why they matter, and deep examples

Point Meaning
Definition UFunc means Universal Function. It is a function that works element by element on NumPy arrays.
Why important They are fast, vectorized, memory-aware, and avoid manual loops.
Examples np.add, np.subtract, np.multiply, np.divide, np.sqrt, np.sin 
arr = np.array([1, 2, 3, 4])

np.add(arr, 10)
np.multiply(arr, 5)
np.sqrt(arr)
np.add.reduce(arr)
np.add.accumulate(arr)
np.multiply.outer([1, 2, 3], [10, 20])
np.where(arr > 2, arr, 0)
  • reduce: combines all elements to one result.
  • accumulate: returns running results step by step.
  • outer: applies the operation to every pair between two arrays.
  • where: chooses values conditionally.
  • out parameter: stores result in a given output array.
  • dtype parameter: controls calculation data type.
  • vectorize and frompyfunc: create custom element-wise function behavior.
Custom UFunc Style

vectorize() and frompyfunc()

def grade_label(x):
    return "Pass" if x >= 40 else "Fail"

vf = np.vectorize(grade_label)
print(vf(np.array([35, 44, 81, 20])))

These tools help apply Python functions across arrays, though built-in NumPy ufuncs are usually faster.

Why NumPy Is Fast

One-line exam answer plus technical reason

  • It stores homogeneous values efficiently in contiguous memory.
  • Many operations are implemented in optimized low-level code.
  • Vectorization removes the overhead of Python-level loops.
  • Broadcasting avoids writing extra manual expansion logic.

Simple process to create a correct chart
