A) Mock Interview Questions and Answer Script

Use this as role-play. Read question first, answer aloud in your own words, then compare with the script and simple example.

Q1. Tell me about yourself for a Python Data Analyst role.

Answer Script: I use Python mainly for data analysis workflows: loading data from CSV, Excel, and SQL, cleaning with Pandas, transforming features, validating with statistics, and building visual reports using Matplotlib and Seaborn. I focus on business questions first, then translate them into reproducible scripts.

Simple example: In one project, I cleaned duplicate orders, calculated monthly profit, and presented the top 3 loss-making categories.

Q2. Difference between loc and iloc?

Answer Script: loc is label-based, so I select rows and columns by name. iloc is integer-position-based, so I select by index position. Also, slicing behavior differs: loc includes end label but iloc excludes end index.

Simple example: df.loc[0, "sales"] picks by column name, while df.iloc[0, 2] picks by third column position.

Q3. How do you handle missing values in real projects?

Answer Script: I first profile null percentages using isnull().sum() and business impact. For critical numeric columns, I use median or group-wise imputation; for categorical, I use mode or explicit labels like "Unknown". If a column has very high nulls and low business value, I drop it with justification.

Simple example: If city has 3% missing values, I fill with "Unknown"; if a column has 85% nulls and low relevance, I remove it.

Q4. Explain a full analysis pipeline in short.

Answer Script: My flow is inspect, clean, transform, analyze, visualize, and summarize actionable insights. I validate each stage with output checks so the final dashboard and conclusions are traceable to actual data operations.

Simple example: I load orders, compute profit = sales - cost, group by region, plot sales trend, then write three actions for the business team.

Q5. What are your top 3 Python interview strengths?

Answer Script: Pandas operations for business analysis, clean and reusable function design, and visualization choices aligned with decision-making rather than decorative charting.

Simple example: I can build a reusable KPI function once and use it for weekly, monthly, and regional reports with small parameter changes.

B) Practical End-to-End Walkthrough

Scenario: Analyze monthly sales and profit by region using raw CSV data.

Step 1: Create and load sample data

import pandas as pd
from io import StringIO

csv_text = """date,order_id,region,category,sales,cost
2026-01-05,101,North,Electronics,45000,32000
2026-01-08,102,South,Home,29000,21000
2026-01-15,103,North,Home,18000,12000
2026-02-03,104,East,Electronics,52000,35000
2026-02-11,105,South,Electronics,41000,30000
2026-02-20,106,North,Home,25000,16000
"""

raw_df = pd.read_csv(StringIO(csv_text), parse_dates=["date"])
print(raw_df.head())
print(raw_df.dtypes)
Expected Output:

0 2026-01-05       101  North  Electronics  45000  32000
1 2026-01-08       102  South         Home  29000  21000
...

Step 2: Feature engineering

df = raw_df.copy()
df["profit"] = df["sales"] - df["cost"]
df["month"] = df["date"].dt.to_period("M").astype(str)
print(df[["order_id", "month", "region", "sales", "cost", "profit"]])
Expected Output: 
   order_id    month region  sales   cost  profit
0       101  2026-01  North  45000  32000   13000
1       102  2026-01  South  29000  21000    8000
...

Step 3: KPI summary

total_sales = df["sales"].sum()
total_profit = df["profit"].sum()
profit_margin = (total_profit / total_sales) * 100

print("Total Sales:", total_sales)
print("Total Profit:", total_profit)
print("Profit Margin %:", round(profit_margin, 2))
Expected Output: 
Total Sales: 210000
Total Profit: 63000
Profit Margin %: 30.0
