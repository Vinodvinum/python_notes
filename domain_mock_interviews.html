Data Analyst Track

Top Mock Questions with Answer Scripts

Q1. How do you approach a new dataset?

Script: I inspect schema and nulls first using head(), info(), and isnull().sum(). Then I clean types, handle missing values with business logic, validate row counts after filters, and only then start KPI and trend analysis.

Simple example: If sales is loaded as text, I convert it with pd.to_numeric() before calculating total revenue.

Q2. How do you explain insights to business users?

Script: I lead with the decision impact, not code. Example: "North contributes 42% of revenue but margin is dropping month over month, so pricing and shipping should be reviewed." Then I show one chart per decision.

Simple example: I show a bar chart for region sales and say, "South is high volume but lower margin, so optimize discount policy there first."

Q3. Difference between descriptive and diagnostic analysis?

Script: Descriptive tells what happened, like monthly sales trend. Diagnostic tells why it happened, like category mix, discount shifts, or return rates causing margin decline.

Simple example: "Sales dropped in March" is descriptive; "Sales dropped because electronics returns increased by 18%" is diagnostic.

Python Developer Track

Core Technical Q&A Scripts

Q1. What is the difference between list and tuple?

Script: List is mutable and best for frequently changing data. Tuple is immutable and safer for fixed records like coordinates or constant config values.

Simple example: Monthly sales targets can be a list because values can change, but GPS coordinates are better stored as a tuple.

Q2. Explain shallow copy vs deep copy.

Script: Shallow copy duplicates only the outer container but nested objects remain shared references. Deep copy recursively clones nested objects so updates do not affect original structures.

Simple example: With a shallow copy of [[1,2],[3,4]], changing inner list value in copy can also change original.

Q3. How do you structure reusable Python code?

Script: I split code into modules by responsibility, write pure functions where possible, add docstrings and type hints, and keep entry logic under if __name__ == "__main__":.

Simple example: I keep cleaning functions in cleaning.py and plotting functions in viz.py, then call both from main.py.

Python Developer Practical

Live Coding Prompt (15-20 mins)

Build a function that reads a list of sales dictionaries and returns a sorted summary by region with error-safe handling.

def summarize_sales(records):
    """Return total sales by region in descending order."""
    summary = {}
    for row in records:
      region = row.get("region", "Unknown")
      sales = float(row.get("sales", 0) or 0)
      summary[region] = summary.get(region, 0) + sales
    return sorted(summary.items(), key=lambda x: x[1], reverse=True)

Follow-up questions expected:

  • What is time complexity?
  • How will you test edge cases?
  • How will you extend for monthly aggregation?
