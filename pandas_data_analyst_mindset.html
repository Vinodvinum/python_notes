Workflow

How a Data Analyst Should Think Before Any Pandas Code

  1. What business question am I solving?
  2. What columns and date range matter for that question?
  3. Can I trust this data? (types, missing values, duplicates)
  4. Which KPI proves the answer? (sum, mean, conversion, trend)
  5. What action should business take based on output?
Rule: Never start with charts. Start with data quality + KPI truth, then visualize.

Deep Example

info() Explained for Analysts

import pandas as pd

df = pd.read_csv("orders.csv")
print(df.info())
Typical output meaning:
  • RangeIndex: 10000 entries -> dataset has 10,000 rows
  • sales 9980 non-null -> 20 missing in sales
  • order_date object -> date is text, not datetime yet

What to do next from this output

  1. Handle missing sales rows: fill or drop based on business rules.
  2. Convert order_date using pd.to_datetime().
  3. Re-run info() and confirm fixes before KPI creation.
Deep Example

describe() Explained for Analysts

print(df[["sales", "profit"]].describe())
Typical output meaning:
  • mean -> average sales/profit
  • 50% -> median, often more stable than mean
  • max much larger than 75% -> possible outlier

What to do next from this output

  1. Use median if outliers are extreme.
  2. Investigate top records using nlargest().
  3. Add boxplot/histogram to validate distribution shape.
Mini Notebook Flow

From Raw Data to Insight in 12 Pandas Steps

# 1) Load
import pandas as pd
df = pd.read_csv("orders.csv")

# 2) First look
df.head(); df.shape; df.info()

# 3) Quality checks
df.isnull().sum(); df.duplicated().sum()

# 4) Fixes
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
df["sales"] = pd.to_numeric(df["sales"], errors="coerce")
df = df.drop_duplicates()

# 5) Missing strategy
df["sales"] = df["sales"].fillna(df["sales"].median())

# 6) Features
df["profit"] = df["sales"] - df["cost"]
df["month"] = df["order_date"].dt.to_period("M")

# 7) KPI
total_sales = df["sales"].sum()
profit_margin = (df["profit"].sum() / total_sales) * 100

# 8) Segment result
state_sales = df.groupby("state")["sales"].sum().sort_values(ascending=False)

# 9) Trend
monthly_sales = df.groupby("month")["sales"].sum()

# 10) Validate
df.describe()

Mindset Training

Data Analyst Mindset: How You Should Think

1) Business-first mindset

Always translate question into metric. Not "run pandas", but "measure conversion drop by channel in Q2".

2) Data quality before dashboard

Never trust first output. Check nulls, duplicates, type mismatch, and date parsing before any KPI or chart.

3) Explainability mindset

Every chart must answer one question. If it looks pretty but does not drive a decision, remove it.

4) Validation mindset

Cross-check one KPI manually each run. Example: compare grouped sum with global total.

5) Action mindset

End every analysis with: what happened, why, and what should be done next.

Interview one-liner: I treat Pandas as a decision engine, not just a coding library. My output always maps to a business action.
