Casting

Detailed Casting Notes

Definition: Casting means converting one data type to another data type.

Explanation: Sometimes input or external data is in string format, but we need integer/float/list operations.

Why we use: To perform correct operations, avoid type errors, and normalize data format.

Applications: User input conversion, API data cleaning, report generation.

# Number casting
num_str = "25"
num_int = int(num_str)
num_float = float(num_str)

# String casting
price = 99
message = "Price is " + str(price)

# Collection casting
nums_tuple = (1, 2, 2, 3)
nums_list = list(nums_tuple)
nums_set = set(nums_tuple)      # removes duplicates
nums_back = tuple(nums_list)

# Dict casting from list of pairs
pairs = [("name", "Vinod"), ("age", 26)]
profile = dict(pairs)
Real-life example: Online form sends age as text "26". You cast to int before checking voting eligibility.
JSON

Detailed JSON Notes

Definition: JSON (JavaScript Object Notation) is a text format for storing and exchanging structured data.

Explanation: Python converts between dictionary/list and JSON string using json.dumps() and json.loads().

Why we use: JSON is lightweight, human-readable, and standard for APIs and config files.

Applications: API request/response, configuration files, logging structured data.

import json

# Python object -> JSON string
data = {"name": "Vinod", "marks": [88, 92], "active": True}
json_text = json.dumps(data, indent=2)
print(json_text)

# JSON string -> Python object
parsed = json.loads(json_text)
print(parsed["name"])

# Save JSON to file
with open("student.json", "w") as f:
    json.dump(data, f, indent=2)

# Read JSON from file
with open("student.json", "r") as f:
    restored = json.load(f)
print(restored)
Real-life example: A mobile app sends user profile data to server in JSON format.
Functions Deep

Detailed Function Notes

Definition: A function is a named reusable block defined with def that can accept input and return output.

Explanation: Functions reduce repeated code and make logic modular and testable.

Why we use: Reusability, readability, easier debugging, clean architecture.

Applications: Validation functions, calculations, formatting utilities, database helpers.

Type Meaning Example
Positional argsOrder matters.add(2, 3)
Keyword argsName-based passing.add(a=2, b=3)
Default argsFallback values.greet(name, city="Bengaluru")
*argsVariable positional arguments.sum_all(*nums)
**kwargsVariable keyword arguments.show_profile(**data)
LambdaSmall one-line function.lambda x: x*x
 
def add(a, b):
    return a + b

def greet(name, city="Bengaluru"):
    return f"Hi {name} from {city}"

def total(*nums):
    return sum(nums)

def profile(**info):
    return info

square = lambda x: x * x

print(add(5, 7))
print(greet("Vinod"))
print(total(1, 2, 3, 4))
print(profile(name="Vinod", role="Learner"))
print(square(6))

Detailed OOP

8.1) Abstraction vs Encapsulation (With Real-Life Example)

Point Abstraction Encapsulation
Meaning Hides implementation details and shows only required features. Bundles data + methods and controls direct access to data.
Focus What an object does. How data is protected.
Used with Abstract classes, interfaces, abstract methods. Private/protected attributes, getter/setter methods.
Real-life example Car dashboard shows speed/fuel only. You do not see engine internals. ATM hides account balance internals and allows changes only through valid operations. 
from abc import ABC, abstractmethod

class Payment(ABC):
    @abstractmethod
    def pay(self, amount):
        pass

class CardPayment(Payment):
    def __init__(self, balance):
        self._balance = balance  # encapsulation

    def pay(self, amount):       # abstraction via common interface
        if amount <= self._balance:
            self._balance -= amount
            return "Payment success"
        return "Insufficient balance"

8.2) Shallow Copy vs Deep Copy (Simple)

Concept Shallow Copy Deep Copy
Nested objects Inner objects are shared. Everything is fully copied.
If inner list changes Original can also change. Original stays safe.
Real-life example Photocopy of house key list but both point to same locker key. Duplicate key set where even locker key is separately duplicated. 
import copy

original = [[1, 2], [3, 4]]
shallow = copy.copy(original)
deep = copy.deepcopy(original)

shallow[0][0] = 999
print("Original after shallow:", original)  # changed

deep[1][1] = 777
print("Original after deep:", original)     # unchanged by deep edit
